## 四、P-E-R 自主模式

### 概念

P-E-R (Planner-Executor-Reflector) 是 ClawAI 的核心架构：

| 角色 | 职责 |
|------|------|
| Planner | AI 分析目标，规划测试步骤 |
| Executor | AI 选择工具，执行测试 |
| Reflector | AI 分析结果，决定下一步 |

### 使用示例

**Python API 调用：**

```python
import asyncio
from src.shared.backend.ai_agent.intelligent_per import IntelligentPERAgent
from src.shared.backend.ai_core.llm_client import AIAgentCore

async def run_test():
    # 初始化 LLM 客户端
    llm = AIAgentCore(provider="deepseek", model="deepseek-chat")
    
    # 创建 P-E-R Agent
    agent = IntelligentPERAgent(
        llm_client=llm,
        tool_executor=your_executor,
        rag_client=your_rag_client,
        max_iterations=5
    )
    
    # 运行测试
    async for event in agent.run(
        target="http://127.0.0.1/dvwa",
        goal="对 DVWA 进行渗透测试"
    ):
        if event["type"] == "task_result":
            print(f"任务: {event['task']}")
            print(f"结果: {event['findings']}")
        
        if event["type"] == "complete":
            print(f"报告: {event['report']}")

asyncio.run(run_test())
```

## 八、常见问题

### Q: 后端启动失败？

```bash
# 检查端口占用
netstat -ano | findstr :8080

# 或更换端口
python -m src.shared.backend.main --port 9090
```

### Q: 前端连接不上后端？

检查 `frontend/.env`:

```bash
VITE_API_URL=http://localhost:8080
```

### Q: 工具执行失败？

1. 检查工具是否在 PATH 中
2. 或配置 `config/tool_paths.yaml`
3. 或使用 Docker 模式

### Q: AI 不执行攻击操作？

某些模型有安全限制，建议：
1. 使用 DeepSeek 或本地模型
2. 配置 `ATTACK_MODE=true` 启用攻击模式
