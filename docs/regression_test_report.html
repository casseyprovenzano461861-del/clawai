Test Methodology

Environment Isolation

Each CVE scenario runs in a dedicated Docker container from the Vulhub library. Containers are isolated per-vulnerability, preventing cross-contamination between test cases.

Skill Execution

Each ClawAI Skill is a self-contained Python module implementing detect(), exploit(), and verify() phases. The regression runner calls skills directly without AI orchestration to ensure deterministic behavior.

Pass Criteria

A skill is PASS if it returns a recognized success token: RCE_SUCCESS, *_DETECTED, JNDI_TRIGGERED with LDAP callback, SQLI_POSSIBLE, or SSTI_DETECTED.

JNDI Callback Validation

For JNDI-based CVEs, an in-network LDAP listener captures outbound callbacks from the vulnerable container. A callback record is required to confirm actual Java JNDI lookup execution.

Protocol Fidelity

Where CVE exploitation requires a specific binary protocol (e.g., IIOP/GIOP for WebLogic, raw deserialization for JBoss), the skill implements the protocol at the socket level rather than relying on HTTP wrappers.

Iterative Refinement

Failing skills are debugged by inspecting raw HTTP/TCP responses against known-good exploit references (Vulhub PoC, ExploitDB, NVD). Each fix is scoped to the minimal change required for correctness.
