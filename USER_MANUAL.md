# ClawAI 用户手册

## 1. 系统概述

ClawAI 是一个基于大模型的自动化渗透测试系统，旨在提高安全测试的效率和准确性。系统整合了先进的AI技术和安全工具，为用户提供了一种高效、全面的安全测试解决方案。

### 1.1 核心功能

- **自动化渗透测试流程**：支持完整的渗透测试生命周期
- **智能工具选择和执行**：基于目标环境智能选择工具
- **实时监控和可视化**：3D攻击链路可视化
- **详细的测试报告生成**：自动生成详细的测试报告
- **多平台支持**：支持Windows和Linux双平台目标环境

### 1.2 系统架构

- **前端**：React 应用，提供用户友好的界面
- **后端**：Python + FastAPI 框架，提供API服务
- **AI 核心**：基于大模型的智能决策系统
- **工具集成**：支持29+安全工具的统一执行

## 2. 快速开始

### 2.1 系统要求

- Python 3.8+
- Node.js 14+
- npm 6+
- 4GB RAM 以上
- 10GB 磁盘空间以上

### 2.2 安装与启动

#### 后端安装

1. 克隆项目仓库
   ```bash
   git clone https://github.com/ClawAI/ClawAI.git
   cd ClawAI
   ```

2. 创建虚拟环境并激活
   ```bash
   python -m venv venv
   # Windows
   venv\Scripts\activate
   # Linux
   source venv/bin/activate
   ```

3. 安装依赖
   ```bash
   pip install -r requirements.txt
   ```

4. 启动后端服务器
   ```bash
   python -m src.shared.backend.main --port 8080
   ```

#### 前端安装

1. 进入前端目录
   ```bash
   cd frontend
   ```

2. 安装依赖
   ```bash
   npm install
   ```

3. 启动前端开发服务器
   ```bash
   npm run dev
   ```

4. 访问前端应用
   打开浏览器，访问 http://localhost:3000

## 3. 系统使用

### 3.1 目标配置

1. 在前端界面的「目标配置」页面，输入目标地址（如 example.com）
2. 选择扫描类型（标准扫描、深度扫描等）
3. 配置扫描参数（如端口范围、扫描深度等）
4. 点击「开始测试」按钮启动渗透测试

### 3.2 测试监控

1. 在「测试监控」页面，实时查看测试进度和阶段状态
2. 查看发现的漏洞列表，按严重程度排序
3. 监控系统状态，包括CPU、内存使用情况
4. 查看工具执行日志，了解详细的测试过程

### 3.3 报告查看

1. 测试完成后，在「报告管理」页面查看详细的测试报告
2. 报告包含漏洞分析、修复建议和风险评估
3. 支持多种报告格式导出（PDF、HTML、JSON）
4. 可以查看历史报告，比较不同测试结果

### 3.4 3D攻击链路可视化

1. 在「攻击链路」页面，查看3D攻击链路图
2. 点击节点查看详细信息，包括漏洞详情和修复建议
3. 调整视角和旋转模式，从不同角度查看攻击链路
4. 导出攻击链路图为图片或PDF

## 4. 高级功能

### 4.1 漏洞验证与PoC生成

1. 在「漏洞管理」页面，选择需要验证的漏洞
2. 点击「验证漏洞」按钮，系统会自动验证漏洞的存在性
3. 验证成功后，点击「生成PoC」按钮，系统会生成漏洞的Proof of Concept
4. 查看PoC详情，包括攻击步骤和预期结果

### 4.2 攻击路径分析

1. 在「攻击路径」页面，点击「分析攻击路径」按钮
2. 系统会分析所有发现的漏洞，生成可能的攻击路径
3. 查看攻击路径的风险评分和详细信息
4. 针对高风险路径，系统会提供相应的防御建议

### 4.3 风险评估

1. 在「风险评估」页面，选择需要评估的漏洞
2. 点击「评估风险」按钮，系统会评估漏洞的风险等级
3. 查看风险评估结果，包括CVSS分数、影响范围和修复建议
4. 系统会根据风险等级提供相应的修复优先级建议

### 4.4 合规性检查

1. 在「合规性」页面，选择需要检查的合规标准（OWASP、PCI DSS、GDPR等）
2. 点击「检查合规性」按钮，系统会检查目标系统的合规性
3. 查看合规性检查结果，包括发现的问题和改进建议
4. 系统会生成合规性报告，支持导出为PDF或HTML

## 5. 工具管理

### 5.1 工具列表

1. 在「工具管理」页面，查看系统集成的所有安全工具
2. 查看工具的详细信息，包括版本、描述和参数
3. 检查工具的安装状态和健康状况

### 5.2 自定义工具

1. 在「工具管理」页面，点击「添加自定义工具」按钮
2. 填写工具名称、路径、描述和参数信息
3. 点击「保存」按钮，系统会添加自定义工具
4. 测试自定义工具的执行情况

### 5.3 工具链

1. 在「工具链」页面，点击「创建工具链」按钮
2. 选择需要包含在工具链中的工具
3. 配置工具的执行顺序和参数
4. 点击「保存」按钮，系统会创建工具链
5. 点击「执行工具链」按钮，系统会按顺序执行工具链中的工具

## 6. 系统配置

### 6.1 环境变量

系统支持通过环境变量进行配置，主要环境变量包括：

- `ENVIRONMENT`：运行环境（development/production）
- `SERVER_HOST`：服务器地址
- `BACKEND_PORT`：服务器端口
- `DATABASE_URL`：数据库URL
- `TOOLS_DIR`：安全工具目录
- `JWT_SECRET_KEY`：JWT密钥
- `SECRET_KEY`：应用密钥

### 6.2 模块配置

系统支持通过 `config/modules.yaml` 文件配置模块的启用状态和参数：

```yaml
modules:
  ai_engine:
    enabled: true
    config:
      llm_provider: "openai"
      default_model: "gpt-4"
      
  data_service:
    enabled: true
    config:
      database_url: "sqlite:///./data/databases/clawai.db"
      
  tool_executor:
    enabled: true
    config:
      tools_dir: "./tools/penetration"
```

### 6.3 RBAC配置

系统支持基于角色的访问控制（RBAC），通过 `config/rbac.json` 文件配置：

```json
{
  "roles": {
    "admin": {
      "permissions": ["TOOL_EXECUTE", "ATTACK_EXECUTE", "USER_MANAGE"]
    },
    "user": {
      "permissions": ["TOOL_EXECUTE", "ATTACK_EXECUTE"]
    }
  }
}
```

## 7. 故障排除

### 7.1 常见问题

1. **应用启动失败**
   - 端口被占用：修改 `BACKEND_PORT` 或停止占用端口的进程
   - 数据库连接失败：检查 `DATABASE_URL` 配置，确保数据库目录可写

2. **工具执行失败**
   - 工具未找到：检查 `TOOLS_DIR` 配置，确保工具目录存在
   - 权限不足：确保工具具有可执行权限

3. **漏洞验证失败**
   - 目标不可达：检查目标地址是否正确，网络连接是否正常
   - 工具配置错误：检查工具的参数配置是否正确

### 7.2 日志管理

系统日志位置：
- 应用日志：`logs/clawai.log`
- 审计日志：`data/audit/`（按日期组织）
- Docker日志：`docker-compose logs -f clawai`

### 7.3 获取帮助

- 查看应用日志：`logs/clawai.log`
- 检查 Docker 日志：`docker-compose logs -f`
- 访问 API 文档：`http://localhost:8000/docs`
- 查看健康状态：`http://localhost:8000/health`

## 9. 常见问题解答

### 9.1 系统相关

**Q: ClawAI 支持哪些操作系统？**
A: ClawAI 支持 Windows 和 Linux 操作系统，目标环境也支持这两种平台。

**Q: ClawAI 需要多少系统资源？**
A: 建议至少 4GB RAM 和 10GB 磁盘空间，具体资源需求取决于测试规模和工具使用情况。

**Q: ClawAI 可以测试哪些类型的目标？**
A: ClawAI 可以测试 Web 应用、网络设备、服务器等多种类型的目标。

### 9.2 工具相关

**Q: ClawAI 集成了哪些安全工具？**
A: ClawAI 集成了 29+ 安全工具，包括 nmap、nikto、sqlmap、nuclei 等。

**Q: 如何添加自定义工具？**
A: 在「工具管理」页面，点击「添加自定义工具」按钮，填写工具信息即可。

**Q: 工具执行失败怎么办？**
A: 检查工具的安装状态和参数配置，查看系统日志了解详细错误信息。

### 9.3 测试相关

**Q: 如何提高漏洞检测率？**
A: 使用深度扫描模式，结合多种工具进行交叉验证，调整工具参数以提高检测灵敏度。

**Q: 如何减少误报？**
A: 使用漏洞验证功能，对发现的漏洞进行验证，结合人工分析判断漏洞的真实性。

**Q: 如何生成高质量的测试报告？**
A: 确保测试过程完整，记录详细的测试步骤和结果，使用系统生成的报告模板。
