## P-E-R 框架 LLM 集成修复（P0任务完成）

**修复内容：**
1. `llm_integration.py`：将 `call_llm()` 从 async 改为同步方法，新增 `call_llm_async()` 异步包装（用 asyncio.to_thread）
2. `planner.py`：移除 `_generate_plan_with_llm` 和 `_dynamic_replan_with_llm` 中危险的 asyncio 嵌套包装，直接同步调用
3. `reflector.py`：移除 `_analyze_with_llm` 和 `_generate_intelligence_with_llm` 中的 asyncio 包装，直接同步调用；修复 `_update_pattern_libraries` 兼容 dict 和 AuditResult
4. `executor.py`：将 `_get_llm_execution_strategy` 改用 `call_llm_async`（非阻塞的 async 调用）
5. `llm_integration.py`：新增 `finish_reason=error` 检测，避免把错误当成功响应

**根本原因：** LLMClient.chat() 是同步方法，而 call_llm 是 async，Planner/Reflector 在同步上下文中用 asyncio.get_event_loop() 嵌套异步调用，在 FastAPI 的已运行事件循环中会死锁。

**验证：** 新增 `tests/test_per_llm_integration.py`，5 个测试全部通过（含 Mock LLM 端到端调用链验证）。
