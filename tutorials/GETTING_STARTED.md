# ClawAI 入门教程

## 1. 系统概述

ClawAI 是一个基于大模型的自动化渗透测试系统，旨在提高安全测试的效率和准确性。本教程将引导您完成 ClawAI 的安装、配置和基本使用。

## 2. 环境要求

- Python 3.8+
- Node.js 14+
- npm 6+
- 4GB RAM 以上
- 10GB 磁盘空间以上

## 3. 安装步骤

### 3.1 克隆项目

```bash
git clone https://github.com/ClawAI/ClawAI.git
cd ClawAI
```

### 3.2 安装后端依赖

```bash
# Windows
python -m venv venv
venv\Scripts\activate

# Linux/Mac
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt
```

### 3.3 安装前端依赖

```bash
cd frontend
npm install
```

## 4. 配置系统

### 4.1 环境变量配置

复制 `.env.example` 文件为 `.env` 并根据需要修改配置：

```bash
cp .env.example .env
```

主要配置项：
- `ENVIRONMENT`：运行环境（development/production）
- `BACKEND_PORT`：后端服务端口
- `DATABASE_URL`：数据库连接字符串
- `TOOLS_DIR`：安全工具目录

### 4.2 工具配置

确保安全工具目录中包含必要的渗透测试工具，如 nmap、sqlmap、nikto 等。

## 5. 启动系统

### 5.1 启动后端服务

```bash
# 在项目根目录
python -m src.shared.backend.main --port 8080
```

### 5.2 启动前端服务

```bash
# 在 frontend 目录
npm run dev
```

### 5.3 访问系统

- 前端：http://localhost:3000
- 后端 API：http://localhost:8080
- API 文档：http://localhost:8080/docs

## 6. 基本使用

### 6.1 目标配置

1. 在前端界面的「目标配置」页面，输入目标地址（如 example.com）
2. 选择扫描类型（标准扫描、深度扫描等）
3. 配置扫描参数（如端口范围、扫描深度等）
4. 点击「开始测试」按钮启动渗透测试

### 6.2 测试监控

1. 在「测试监控」页面，实时查看测试进度和阶段状态
2. 查看发现的漏洞列表，按严重程度排序
3. 监控系统状态，包括CPU、内存使用情况
4. 查看工具执行日志，了解详细的测试过程

### 6.3 报告查看

1. 测试完成后，在「报告管理」页面查看详细的测试报告
2. 报告包含漏洞分析、修复建议和风险评估
3. 支持多种报告格式导出（PDF、HTML、JSON）
4. 可以查看历史报告，比较不同测试结果

## 7. 高级功能

### 7.1 漏洞验证与PoC生成

1. 在「漏洞管理」页面，选择需要验证的漏洞
2. 点击「验证漏洞」按钮，系统会自动验证漏洞的存在性
3. 验证成功后，点击「生成PoC」按钮，系统会生成漏洞的Proof of Concept
4. 查看PoC详情，包括攻击步骤和预期结果

### 7.2 攻击路径分析

1. 在「攻击路径」页面，点击「分析攻击路径」按钮
2. 系统会分析所有发现的漏洞，生成可能的攻击路径
3. 查看攻击路径的风险评分和详细信息
4. 针对高风险路径，系统会提供相应的防御建议

### 7.3 风险评估

1. 在「风险评估」页面，选择需要评估的漏洞
2. 点击「评估风险」按钮，系统会评估漏洞的风险等级
3. 查看风险评估结果，包括CVSS分数、影响范围和修复建议
4. 系统会根据风险等级提供相应的修复优先级建议

### 7.4 合规性检查

1. 在「合规性」页面，选择需要检查的合规标准（OWASP、PCI DSS、GDPR等）
2. 点击「检查合规性」按钮，系统会检查目标系统的合规性
3. 查看合规性检查结果，包括发现的问题和改进建议
4. 系统会生成合规性报告，支持导出为PDF或HTML

## 8. 工具管理

### 8.1 工具列表

1. 在「工具管理」页面，查看系统集成的所有安全工具
2. 查看工具的详细信息，包括版本、描述和参数
3. 检查工具的安装状态和健康状况

### 8.2 自定义工具

1. 在「工具管理」页面，点击「添加自定义工具」按钮
2. 填写工具名称、路径、描述和参数信息
3. 点击「保存」按钮，系统会添加自定义工具
4. 测试自定义工具的执行情况

### 8.3 工具链

1. 在「工具链」页面，点击「创建工具链」按钮
2. 选择需要包含在工具链中的工具
3. 配置工具的执行顺序和参数
4. 点击「保存」按钮，系统会创建工具链
5. 点击「执行工具链」按钮，系统会按顺序执行工具链中的工具

## 9. 常见问题

### 9.1 服务启动失败

- **端口被占用**：修改 `BACKEND_PORT` 环境变量或停止占用端口的进程
- **数据库连接失败**：检查 `DATABASE_URL` 配置，确保数据库目录可写
- **依赖缺失**：重新安装依赖 `pip install -r requirements.txt`

### 9.2 工具执行失败

- **工具未找到**：检查 `TOOLS_DIR` 配置，确保工具目录存在
- **权限不足**：确保工具具有可执行权限
- **参数错误**：检查工具参数配置是否正确

### 9.3 漏洞检测问题

- **误报**：使用漏洞验证功能确认漏洞的真实性
- **漏报**：使用深度扫描模式，调整工具参数以提高检测灵敏度
- **检测速度慢**：优化系统资源分配，调整并行执行设置
