# ClawAI 启动问题解决方案

## 🚨 主要问题

### 问题1：端口8000被占用
**错误信息**：
```
ERROR: [Errno 10048] error while attempting to bind on address ('0.0.0.0', 8000): [winerror 10048] 通常每个套接字地址(协议/网络地址/端口)只允许使用一次。
```

**原因**：
- 另一个Python进程（可能是之前的ClawAI实例）占用了8000端口
- 其他应用（如其他Web服务）使用了该端口

**解决方案**：
1. **使用智能启动脚本**（推荐）：
   ```bash
   start_clawai.bat
   ```
   自动检测可用端口（8000-8010）

2. **手动指定端口**：
   ```bash
   python run.py --port 8001
   ```

3. **停止占用进程**：
   ```bash
   # 查看占用8000端口的进程
   netstat -ano | findstr :8000
   
   # 停止该进程（PID来自上条命令）
   taskkill /PID [PID] /F
   ```

### 问题2：缺少neo4j模块
**警告信息**：
```
WARNING - 知识图谱API路由导入失败: No module named 'neo4j'
```

**影响**：
- 知识图谱功能不可用
- 其他功能正常（有回退机制）

**解决方案**（可选）：
```bash
pip install neo4j
```

### 问题3：llm_agent模块缺失
**警告信息**：
```
WARNING - 无法导入LLM集成器: No module named 'llm_agent'
```

**影响**：
- AI引擎功能受限
- 基本AI功能仍然可用

## 🚀 推荐的启动方式

### 方式一：智能启动（最推荐）
```bash
start_clawai.bat
```
- 自动检测端口占用
- 自动管理虚拟环境
- 自动安装依赖

### 方式二：Docker启动
```bash
scripts\start_docker.bat
```
选择选项1，然后访问：
- 后端API: http://localhost:8000
- Neo4j: http://localhost:7474 (用户: neo4j, 密码: password)

### 方式三：手动启动
```bash
# 1. 创建虚拟环境
python -m venv .venv
.venv\Scripts\activate

# 2. 安装依赖
pip install -r requirements.txt

# 3. 使用可用端口启动
python run.py --port 8001
```

## 🔧 快速故障排除

### 1. 端口问题
```bash
# 检查端口占用
netstat -ano | findstr :8000

# 如果8000被占用，使用其他端口
python run.py --port 8001
```

### 2. 依赖问题
```bash
# 重新安装依赖
pip install -r requirements.txt --upgrade

# 安装缺失的模块
pip install neo4j
```

### 3. 数据库问题
```bash
# 确保数据库目录存在
mkdir -p data\databases

# 检查数据库文件权限
```

## 📋 验证启动成功

1. **检查服务状态**：
   ```bash
   curl http://localhost:8001/health
   ```
   应返回：`{"status":"healthy","version":"2.0.0"}`

2. **访问API文档**：
   浏览器打开：http://localhost:8001/docs

3. **查看日志**：
   启动时会显示详细的初始化日志，确认没有错误。

## ⚡ 一键修复脚本

如果问题持续，运行：
```bash
# 停止所有可能的ClawAI进程
python -c "import os; os.system('taskkill /F /IM python.exe')" 2>nul

# 清理后重启
start_clawai.bat
```

## 📞 紧急联系方式

如果以上方法都无法解决问题：
1. 查看 `logs/clawai.log` 获取详细错误信息
2. 检查Windows防火墙设置
3. 确保Python 3.11+已正确安装

---

**总结**：主要问题是端口8000被占用，使用 `start_clawai.bat` 可自动解决。
