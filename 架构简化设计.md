# ClawAI 架构简化设计 (P0-11: 微服务降级为模块化单体)

## 当前架构分析

### 1. 现有微服务结构

项目目前存在两套并行的架构：

1. **独立微服务** (`services/`目录):
   - `ai-engine/`: AI引擎服务 (端口: 8001)
     - `/health`, `/metrics`
     - `/api/v1/llm/configs`, `/api/v1/llm/analyze`, `/api/v1/llm/generate-plan`
     - `/api/v1/ai/analyze`, `/api/v1/ai/generate-plan`, `/api/v1/ai/execute-skill`
     - `/api/v1/skills`, `/api/v1/configs/{config_name}`
   - `api-gateway/`: API网关服务 (端口: 8000)
     - `/health`, `/metrics`
     - `/{path:path}` (代理路由)
   - `data-service/`: 数据服务 (端口: 8002)
     - `/health`
     - `/api/v1/users/register`, `/api/v1/users/login`
     - `/api/v1/projects`, `/api/v1/projects/{project_id}`
     - `/api/v1/results`, `/api/v1/experiments`
   - `tool-executor/`: 工具执行服务 (端口: 8003)
     - `/health`
     - `/api/v1/tools/available`, `/api/v1/tools/execute`
     - `/api/v1/tasks/{task_id}`

2. **模块化代码库** (`src/`目录):
   - `shared/backend/main.py`: 主FastAPI应用 (当前通过run.py启动)
     - `/`, `/health`
     - `/tools`, `/tools/health`
     - `/attack`, `/tools/execute`
   - 独立的模块目录:
     - `ai_engine/`: 包含独立的FastAPI应用
     - `api_gateway/`: 包含独立的FastAPI应用  
     - `data_service/`: 包含独立的FastAPI应用
     - `tool_executor/`: 包含独立的FastAPI应用

### 2. 问题识别

1. **架构混乱**: 两套并行架构，难以维护
2. **代码重复**: 相同功能在不同地方实现
3. **启动复杂**: 需要启动多个服务
4. **依赖管理复杂**: 多个独立的requirements.txt文件
5. **调试困难**: 分布式问题追踪复杂

## 技术细节

### 1. 模块发现和加载

```python
# 模块加载器
class ModuleLoader:
    def __init__(self, app: FastAPI, config: Config):
        self.app = app
        self.config = config
        self.modules = {}
    
    def load_module(self, module_name: str, module_class):
        """动态加载模块"""
        if self.config.get(f"modules.{module_name}.enabled", True):
            module = module_class(self.app, self.config)
            module.startup()
            module.register_dependencies()
            module.register_routes()
            self.modules[module_name] = module
            return module
        return None
```

### 2. 数据库连接共享

使用SQLAlchemy连接池，所有模块共享数据库连接：

```python
# 数据库管理器 (单例)
class DatabaseManager:
    _instance = None
    
    def __init__(self, config: Config):
        self.engine = create_engine(config.database.url)
        self.SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=self.engine)
    
    @classmethod
    def get_instance(cls, config: Optional[Config] = None):
        if cls._instance is None and config is not None:
            cls._instance = cls(config)
        return cls._instance
```

### 3. 事件总线（模块间通信）

实现简单的事件总线，用于模块间解耦通信：

```python
# 事件总线
class EventBus:
    def __init__(self):
        self.subscribers = defaultdict(list)
    
    def subscribe(self, event_type: str, callback: Callable):
        self.subscribers[event_type].append(callback)
    
    def publish(self, event_type: str, data: Any):
        for callback in self.subscribers.get(event_type, []):
            callback(data)
```

## 预期收益

1. **简化部署**: 单个容器/进程，无需服务发现
2. **降低运维复杂度**: 监控、日志、调试统一
3. **提高开发效率**: 代码集中，便于理解和修改
4. **减少资源占用**: 共享数据库连接、缓存等资源
5. **保持扩展性**: 模块化设计，未来可重新拆分为微服务

## 风险与缓解

1. **风险**: 整合过程中可能引入bug
   **缓解**: 分阶段重构，每个模块单独测试

2. **风险**: 性能问题（单个进程承载所有功能）
   **缓解**: 异步处理，性能监控，未来可按需拆分

3. **风险**: API兼容性破坏
   **缓解**: 保持现有API路径，逐步迁移
